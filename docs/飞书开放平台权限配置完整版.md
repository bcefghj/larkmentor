# 飞书开放平台权限配置完整版

> **Agent-Pilot v2 需要的所有飞书权限和配置项清单**  
> 配置时间约 **15 分钟**，按以下步骤逐一完成即可。

---

## 第一步：创建自建应用

1. 打开 [飞书开放平台](https://open.feishu.cn/app)，用你的飞书账号登录
2. 点击右上角 **「创建应用」**
3. 选择 **「自建应用」**
4. 填写应用信息：
   - 应用名称：`LarkMentor Agent-Pilot`
   - 应用描述：`基于 IM 的办公协同智能助手 - 从对话到演示稿一键智能闭环`
   - 上传图标：可选
5. 点击 **「确定创建」**

---

## 第二步：获取应用凭证

进入应用后，左侧菜单 **「凭证与基础信息」**：

- **App ID**（形如 `cli_xxxxxxxxx`）
- **App Secret**（点击「查看」复制，形如 `xxxxxxxxxxxxxxxx`）

把这两个值填进你的 `.env` 文件：

```bash
FEISHU_APP_ID=cli_xxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxx
```

---

## 第三步：开启机器人能力

1. 左侧菜单 **「应用功能」→「机器人」**
2. 点击 **「开启机器人」**
3. **消息卡片请求网址**：留空（我们使用 WebSocket 长连接）
4. 设置机器人头像和名称（可选）

---

## 第四步：申请 API 权限

左侧菜单 **「权限管理」**，在搜索框中依次搜索并申请以下权限：

### 🔴 必须权限（Agent-Pilot 核心功能）

| 权限标识 | 权限名称 | 用途 |
|----------|----------|------|
| `im:message` | 获取与发送单聊、群组消息 | 接收用户指令，发送响应 |
| `im:message:send_as_bot` | 以应用身份发送消息 | Bot 主动推送卡片消息 |
| `im:chat` | 获取群组信息 | 识别群聊上下文 |
| `im:chat:readonly` | 获取群组信息（只读） | 读取群基本信息 |
| `contact:user.base:readonly` | 获取通讯录基础信息 | 解析用户姓名和头像 |
| `docx:document` | 创建、编辑和删除文档 | **核心**：创建产品方案文档 |
| `docx:document:readonly` | 获取文档内容 | 读取已有文档内容 |

> **注意**：您的截图显示 `bitable:app` 等权限已开通 ✅，但请确认以下权限状态

### 🟡 重要权限（Agent-Pilot 高级功能）

| 权限标识 | 权限名称 | 用途 |
|----------|----------|------|
| `bitable:app` | 创建、编辑、删除多维表格 | 结构化数据存储 |
| `bitable:app:readonly` | 获取多维表格信息 | 读取数据表内容 |
| `minutes:minutes` | 获取妙记内容 | **语音转写**：会议纪要集成 |
| `minutes:minutes:readonly` | 获取妙记信息 | 语音输入转文字 |
| `board:whiteboard:node:create` | 创建画板节点 | **架构图**：飞书画板双写 |
| `board:whiteboard:node:read` | 查看画板节点 | 画布内容同步 |
| `board:whiteboard:node:update` | 更新画板节点 | 画布内容编辑 |
| `board:whiteboard:node:delete` | 删除画板节点 | 画布内容管理 |

### 🟢 可选权限（锦上添花）

| 权限标识 | 权限名称 | 用途 |
|----------|----------|------|
| `calendar:calendar` | 日历写入权限 | 自动创建评审会议 |
| `calendar:calendar:readonly` | 日历读取权限 | 查询忙闲状态 |
| `task:task` | 任务管理权限 | 飞书任务 v2 集成 |
| `task:task:readonly` | 任务读取权限 | 读取任务状态 |
| `wiki:wiki:readonly` | 知识库读取权限 | Wiki 知识检索 |
| `drive:drive:readonly` | 云文档读取权限 | 文件链接解析 |

---

## 第五步：配置事件订阅

左侧菜单 **「开发配置」→「事件与回调」**：

### 1. 选择接收方式
**重要**：选择 **「使用长连接接收事件」**（不要选 HTTP 回调）

### 2. 订阅必需事件

| 事件标识 | 事件名称 | 用途 |
|----------|----------|------|
| `im.message.receive_v1` | 接收消息 | 接收用户的 `/pilot` 指令 |
| `card.action.trigger` | 卡片交互 | 处理卡片按钮点击 |

### 3. 订阅可选事件

| 事件标识 | 事件名称 | 用途 |
|----------|----------|------|
| `im.chat.member.bot.added_v1` | 机器人进群 | 自动发送使用指南 |
| `im.chat.member.bot.deleted_v1` | 机器人退群 | 清理相关数据 |

---

## 第六步：发布应用版本

**重要**：每次修改权限或事件配置后，都需要发布新版本！

1. 左侧菜单 **「应用发布」→「版本管理与发布」**
2. 点击 **「创建版本」**
3. 填写版本信息：
   - 版本号：`1.0.0`
   - 更新说明：`Agent-Pilot v2 初始版本`
4. 点击 **「提交审核」**

> **审核时长**：
> - 测试企业（个人开发者）：**立即通过**
> - 正式企业：需管理员审批（通常 1-2 工作日）

---

## 第七步：获取火山方舟 API Key

Agent-Pilot 使用火山方舟 Doubao 模型：

1. 注册 [火山引擎账号](https://console.volcengine.com/)
2. 进入 [方舟控制台](https://console.volcengine.com/ark)
3. 左侧 **「API 管理」→「API Key 管理」**
4. 点击 **「新建 API Key」**
5. 复制 API Key（形如 `ark-xxxxxxxxxxxx`）

填入 `.env`：
```bash
ARK_API_KEY=ark-xxxxxxxxxxxx
ARK_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
ARK_MODEL=doubao-pro-32k
```

```bash
# === 飞书开放平台（必填） ===
FEISHU_APP_ID=cli_xxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxx

# === 火山方舟 LLM（必填） ===
ARK_API_KEY=ark-xxxxxxxxxxxx

ARK_MODEL=doubao-pro-32k

# === Agent-Pilot Dashboard（必填） ===
DASHBOARD_PORT=8001
DASHBOARD_HOST=0.0.0.0

# === 飞书多维表格（可选） ===
BITABLE_APP_TOKEN=
BITABLE_TABLE_ID=

# === MCP 服务（可选） ===
MCP_PORT=8767
MCP_HOST=127.0.0.1

# === 其他配置 ===
LOG_LEVEL=INFO
DATA_DIR=./data
ARTIFACTS_DIR=./data/pilot_artifacts
```

## 常见问题

| 问题症状 | 可能原因 | 解决方案 |
|----------|----------|----------|
| Bot 不响应 | 事件未订阅或版本未发布 | 检查事件订阅 → 重新发布版本 |
| 权限报错 `99991403` | API 权限未申请 | 补充申请权限 → 发布新版本 |
| WebSocket 连接失败 | App ID/Secret 错误 | 检查 `.env` 配置 |
| 创建文档失败 | `docx:document` 权限缺失 | 申请文档权限 → 发布版本 |
| 语音转写不可用 | `minutes:minutes` 权限缺失 | 申请妙记权限 → 发布版本 |
| 白板创建失败 | `board:whiteboard` 权限缺失 | 申请画板权限 → 发布版本 |

**调试技巧**：
- 查看日志：`tail -f logs/larkmentor.log`
- 测试 API：Dashboard → `/api/pilot/scenarios`
- 验证权限：飞书开放平台 → 权限管理 → 已获得权限

## 权限申请优先级

**如果时间有限**，按以下优先级申请：

### P0（必须，核心功能无法运行）
- `im:message` + `im:message:send_as_bot`
- `im:chat` + `contact:user.base:readonly`
- `docx:document` + `docx:document:readonly`

### P1（重要，影响核心体验）
- `bitable:app` + `board:whiteboard`
- `minutes:minutes`（语音转写）

### P2（可选，锦上添花）
- `calendar:calendar` + `task:task` + `wiki:wiki:readonly`

**建议**：先申请 P0 权限快速验证，再逐步补齐 P1、P2 权限。
